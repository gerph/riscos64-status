# Improved testing of the status and C library repos

If you've been following the [64-bit status repository](https://github.com/gerph/riscos64-status), you'll have seen a few new modules turn up recently, and may have noticed the new tests.

If not, you can see these in the most recent test runs under 'Actions', which show all the modules and absolutes being tested to check that they're doing the right thing (where 'right thing' in this case is 'loading and being killed without crashing', which is a reasonable smoketest). For example, if you look at https://github.com/gerph/riscos64-status/actions/runs/16250176536 you should see all the test results present. And you can go into the logs themselves to find the test output if you're interested.

In future, this can probably be extended, but it's really only a test to check that the modules haven't become broken by anything.

The [RISC OS 64 C library repository](https://github.com/gerph/riscos64-simple-binaries/) has also been updated with a similar set of tests, to check that the library is working correctly. For example, you can see the tests in the Actions for that repository - https://github.com/gerph/riscos64-simple-binaries/actions/runs/16241295376

Testing is obviously important for all software, so having the test functionality present in the CI is really useful to ensure that things don't become broken at an inopportune time!
