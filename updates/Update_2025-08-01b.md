# C library repository rename

I've updated the [riscos64-status repository](https://github.com/gerph/riscos64-status) with some improved terminology so that the progress of each of the components can be described in a more structured way. There's also a few new RISC OS 64 binaries present as well.

Since I posted last...

* The state of each component is now using standardised terms, so that a report can be produced for the state of each area - https://github.com/gerph/riscos64-status/actions/runs/16657076915 shows the test results and a report on the functional state of the components being worked on.
* New `MessageTrans` module has been added.
* `Debugger` updated with recent work to add the constant comments.
* New binaries added for: `Perl`, `CMunge`, `Kitten`, `VTranslate`, `Bison`, `Flex`, `Sed`.
* New `SystemVarsDefaults` module created, which manages the initial kernel-set variables.
* Notes for tools updated to indicate that `tcc` is building and can [compile simple things](https://asciinema.org/a/730754)
