# C library repository rename

When the `riscos64-simple-binaries` repository was created, in July last year, it was merely a place that I could put the simple `cog` demonstration and the library that made it work so that it could be made available to people after my 64-bit presentation. Since then, it has grown significantly, and now comprises the C library that I use when building all the 64-bit RISC OS components.

I have mentioned in other posts that it's been increasingly inaccurately named, so I've decided to change its name to `riscos64-clib`. This is what the repository is for, so it's best to accept it (smile). If the repository is used by its old name, Git/GitHub will silently handle the redirection for you.

You can find the repository here: https://github.com/gerph/riscos64-clib/

The repository now includes a Wiki which contains a changelog for the repository - https://github.com/gerph/riscos64-clib/wiki/Changes.

I've also created release 0.17, which is largely updates and fixes necessary for <code>tcc</code> to compile and run.
