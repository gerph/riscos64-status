# Status update (November 2025)

It's been about 4 weeks since I did a status update on the 64-bit RISC OS work.

A lot of distractions over this period, getting married and settling into the new job, but there have been some interesting developments...

* The `SystemVars` module is pretty much complete now, supporting all the operations and working on both the 32bit and 64bit systems. See https://github.com/gerph/riscos-systemvars for the details.
* The Wimp Templates subcomponent is currently being worked on as part of the live coding.
* The notes in the `riscos64-status` repository have had more updates, with more component details updated.
    * The BASIC module has been expanded based on notes from Steve Revill.
    * Each component has also been updated to include its expected state during each phase, which makes it clear what can be expected during development.
    * Phases now have little percentage completion icons to make it clear how far they have progressed. For example: <img src="https://gist.githubusercontent.com/gerph/c26e8457269506554ec1f7533d2f9aed/raw/Progress-1.svg">
    * New Languages page gives the explicit progress on language ports. See https://github.com/gerph/riscos64-status/wiki/Languages for details.
    * See https://github.com/gerph/riscos64-status/wiki for the details and https://github.com/gerph/riscos64-status/wiki/Progress for the overall progress.
* Simple port of `PicoC` which allows interactive C calls (including SWI calls), and makes testing in the 64-bit environment possible without a good BASIC interpreter. See https://gitlab.com/gerph/picoc for details.
* Rudimentary support for Fortran in the build tools, though it's not tested much. See https://asciinema.org/a/753942 for the first 'hello world'.
* The C library has had a lot of improvements made, mostly in support of other components that have needed them - largely floating point, but a variety of other functions. See https://github.com/gerph/riscos64-clib for the sources and release.

On the 17th I'll be doing a presentation to ROUGOL about the progress with RISC OS Pyromaniac, 64-bit development, tooling and related things, over Zoom.

Anyhow, that’s where things are for me. You can always follow the repositories to see what’s being done.

As usual, if anyone wants to contribute to these, please speak up.
