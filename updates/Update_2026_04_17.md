# Status Update (April 2026)

It's about 4 weeks since the last update on 64-bit progress, and apparently there's been quite a bit of work going on.

* Pyromaniac work continues.
    * Experimental Python 3 support work is now actually passing its tests.
      See tests passing: https://share.gerph.org/s/hnrlhvL3tUddl1s. See the desktop running in with Python 3, Unicorn 2, Capstone 5: https://share.gerph.org/s/bJSrhkkddMqQMij. They're not thrilling, 'cos they're just running.
    * New NetStatus module added to the system (Just an implementation of the classic module).
    * New JUnitXML module. See https://github.com/gerph/riscos-junitxml-module-ai for details.
* C version of the EasySocket module is looking for testers. See https://github.com/gerph/riscos-make-a-module for details.
* New JUnitXML module in C (as well as the PyModule implementation). See https://www.riscosopen.org/forum/forums/1/topics/20351 for details.
* Live Coding continues.
    * Recently finished implementin the *GOS command from the UtilityModule for 32-bit and 64-bit. See https://github.com/gerph/riscos-gos for the source and release, and https://www.youtube.com/playlist?list=PLVVIu906Y7rF2ztisbUa0dVTV6MvNS9si for the playlist.
    * WindowManager integration of the C template loading was completed and is available. See https://github.com/gerph/riscos5-wimp for details.
* Another release of the RISC OS build environment.
    * Build environment improves much of the internal help, adds AI agent support. See https://hub.docker.com/repository/docker/gerph/riscos-build/general for more details.
    * Series of short form videos explain the RISC OS build environment and demonstrate how to do simple tasks. https://www.youtube.com/playlist?list=PLVVIu906Y7rETZ8uCBVwjLt5HPlDG7xnn
* Release of agent skills for RISC OS. See https://github.com/gerph/riscos-agent-skills for details.
* Release of GitHub action for building through the build service. See https://github.com/gerph/riscos-build-service-action for details. Also on the GitHub marketplace: https://github.com/marketplace?query=risc+os
* Example repositories created:
    * Example of Vector trapping with CMunge produced. See https://github.com/gerph/riscos-vectortrap-example for details.
    * Simple example of building C code with the RISC OS Build service. See https://github.com/gerph/riscos-build-c-example for details. Notionally similar to the earlier BASIC code example for the RISC OS build service - https://github.com/gerph/riscos-build-basic-example.
    * Example CLI tool, created as part of the short video series. See https://github.com/gerph/example for details.
    * Example module, created as part of the short video series. See https://github.com/gerph/example-module for details.
* 64-bit module progress continues.
    * TimerManager C module prototype created. See https://github.com/gerph/riscos-timermanager for details.
    * OS_FSControl implementation of the commands (simple and multi-file) is nearly complete.
    * NetStatus module C implementation is complete. See https://github.com/gerph/riscos-netstatus for details.
    * Memory management work for the Kernel has been started.
    * VideoSW C module stub created. Still too experimental.
    * OSSWIs module improved to cover almost all the SWIs. See https://github.com/gerph/riscos64-status for the modules. In particular, OS_PrettyPrint, OS_SubstituteArgs, OS_ReadArgs now have implementations. See https://github.com/gerph/riscos64-status/wiki/Module_OSSWIs for functionality.
    * EvaluateExpression module implemented, covering all the cases. See https://github.com/gerph/riscos64-status for the modules. See https://github.com/gerph/riscos64-status/wiki/Module_EvaluateExpression for functionality.
    * Draw module stub created. Still experimental.
    * NVRAMHW module prototype created. See https://github.com/gerph/riscos-nvramhw for details.
    * Prototype Joystick module implementation created. See https://github.com/gerph/riscos-joystick-ai for details.
    * Wimp validation strings parser started. Very stubby, and doesn't include many options.
* The general riscos64-status repository (https://github.com/gerph/riscos64-status) has been kept up to date with these changes as they happened.
    * But GitHub appear to have broken all the nice rainbow status charts, so they just appear in green now. There will be work to ensure that these appear better in the future.
    * All the status updates have been posted on the RISC OS 64 wiki: https://github.com/gerph/riscos64-status/wiki/Updates. Hopefully this will mean that they can be found more easily.

In other news, ROOL have claimed to have a TerritoryManager in C, but I've not had any reply to a request for where this is. That would at least remove another module from those that need to be implemented.
