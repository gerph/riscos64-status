# Status update (March 2026)

It's about 6 weeks since the last update on the 64-bit progress, although I've had a lot of real work and a week's holiday, so it's been a little more light this time. Here's the update...

* WindowManager text rendering was completed as part of the live coding. See https://github.com/gerph/riscos-wimp-textrender for details.
* Floating point updates for the 64-bit RISC OS C library. See https://github.com/gerph/riscos64-clib for details.
* WindowManager integration of the C templates code is being done as part of the live coding. See https://github.com/gerph/riscos5-wimp/tree/c-templates-experiment for detais.
* Improvements to the GContext library for graphics context to support the Wimp text rendering. See https://github.com/gerph/riscos-gcontext for details.
* Built the RPCEmu Docker image from David Ramsden's integration of Andrew Timmins' debugger fork.
    * See https://hub.docker.com/repository/docker/gerph/rpcemu-5/ and https://hub.docker.com/repository/docker/gerph/rpcemu-3.7/general for details (the 'debug' tag contains the new image).
    * [Screenshot](See https://share.gerph.org/s/W5MShIXJJ3Xj9Fn)
* Improvements to the hourglass module.
    * Default colour variants are now supported, allowing the module build to be reconfigured.
    * Colour SWIs are now implemented.
    * New hourglass 'jimphic' module created using a circular pattern.
    * See https://github.com/gerph/riscos-hourglass-maker for details.
* Initial RISC OS Pyromaniac support for filing systems has been merged.
    * Filing systems can be registered and deregistered.
    * Buffered filing systems work well at reading files.
    * Testing with CDFS, PipeFS, ResourceFS and the SystemDevices works/shows lots of promise.
    * [Video](https://asciinema.org/a/793001) of the first filing system operations.

The overall status of the components can be found in the [RISC OS 64 Status repository](https://github.com/gerph/riscos64-status/wiki)

You can always follow the repositories to see what’s being done. The RISC OS Status repository also has a `Discussions` section for general discussions on the progress of 64-bit work.

The sponsorship programme is something that GitHub offers to allow people to provide contributions towards the RISC OS 64 project. If you like what’s being done, and want to encourage more, feel free to sponsor.

And of course, if you want to get involved in working on any of the conversion to C of components, please speak up.
