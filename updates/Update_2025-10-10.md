# Status update (October 2025)

A quick update on the last 3 weeks of 64-bit RISC OS work.

It's not been an especially productive period, as my attention has been elsewhere for a lot of the time, but there are a few interesting things that have been achieved...

* The `SystemVars` module has been part of the live coding and is developing well. Creating, reading, updating, deleting and enumerating variables now works in the internal routines, and we can set and read all the main variable types. The only type that's not yet implemented is the code variable type, but I believe this will be relatively simple to implement. See https://github.com/gerph/riscos-systemvars for the details.
* `ModGen` is now working and can create 64-bit modules to register resources.
* I've started on `ResourceFS`, and the internals are pretty much working - registration, deregistration and services are all implemented. None of the filesystem itself is, but internally the file details can be read and enumerated.
* I've created a new repository in GitHub for the old Acorn example `FS-in-C` sources, and updated it to build with the tools. There's still some assembler in there - it does not use the CMHG header entries, so that will need fixing, but the base interface is there. See https://github.com/gerph/riscos-fsinc-example for the example code.
* The notes in the `riscos64-status` repository have been updated to include more information about the functionality that is present in each module. This should make it easier to tick off features as they go. See https://github.com/gerph/riscos64-status/wiki for the details and https://github.com/gerph/riscos64-status/wiki/Progress for the overall progress.
* A number of other modules were built for 64-bit, including `VideoTTX` which seemed to work ok.
* `PocketPy`, a small version of Python, was ported to RISC OS 64 and appears to be working ok. See https://github.com/gerph/riscos64-pocketpy for the source, https://asciinema.org/a/745457 for a video of it running.
* `Berry`, a lightweight scripting language, has been ported to RISC OS 64, and appears to start its command line ok. See https://github.com/gerph/riscos64-berry for the sources.
* C library improvements to lots of FP functions and a few other standard library functions so that it could be used with `PocketPy`. See https://github.com/gerph/riscos64-clib for the sources and release.
* `ArchieSDK`, a C library and environment for Archimedes class machines and later has been ported to RISC OS 64, making the second C library which is available for building 64-bit components (not tested with modules at all). See https://gitlab.com/gerph/riscos64-archie-sdk for the sources.

Anyhow, that’s where things are for me. You can always follow the repositories to see what’s being done.

I’ve also tried to ensure that components which build as 64-bit are marked explicitly with the ‘riscos64’ label on GitHub.

As usual, if anyone wants to contribute to these, please speak up.
