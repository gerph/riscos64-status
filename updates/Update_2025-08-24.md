# Status update (improved plans and phases).

I've updated the [riscos64-status repository](https://github.com/gerph/riscos64-status/) with a bit of a restructure. I've firmed up a lot of my plans now, so I've updated the texts on the repository and moved things around. Also, a few more updates to components as well.

Since I posted last...

* The main repository now has some better [descriptions of the plans](https://github.com/gerph/riscos64-status/wiki/Planning)
* The work that I'm trying to do is broken down into phases, which are vaguely described, and which should all show progress. For example, Phase 1 is where we are right now, and includes some very simple stubs and a few small implementations of modules: https://github.com/gerph/riscos64-status/wiki/Phase-1
* The progress on each of the phases is now given some nice colour-coded charts to show where we're up to. Yes, there's a lot to do... https://github.com/gerph/riscos64-status/wiki/Progress and yes, I've spent a bit of time trying to make pretty charts and trying to work out how things fit together
* `Debugger` updated with the AArch64 disassembler from recent live coding - AArch64 support and a few more bugs squashed.
* Stub implementation of the `ARM` module is now in the binaries.
* `BootCommands` has been built and seems to work - lots of updates to the `riscos64-oslib` repository to ensure that OSLib is working as expected.
* `DrawFile` has been built, but it crashes when you try to use it.

Independently, the [C library](https://github.com/gerph/riscos64-clib) has been updated to handle more of the standard library, and in particular, command line redirection now works. There's a stub implementation of `system` which isn't much use, but at least it allows some components to work. The relocation tables have been a constant source of problems in the static binaries, but I think I've squashed all the deficiencies now. A port of the Chibicc for AArch64 has also been created and seems to be able to build simple C code ([chibicc-aarch64](https://github.com/xhackerustc/chibicc-aarch64) is the source). Unfortunately, it's not sufficient for too much.

If anyone wants to contribute some work towards building new modules, please speak up :-)
