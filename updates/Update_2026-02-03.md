# Status update (February 2026)

We've had a month and a half since the last status update on the 64-bit RISC OS work, so here we go...

* Kernel introspection calls were given some stubbed entry points as part of the live coding work. See https://github.com/gerph/riscos-kernel-introspection for details.
* Fixes for the Hourglass Maker have been applied (it seemed to always turn the pointer off!). See https://github.com/gerph/riscos-hourglass-maker for details.
* Wimp text rendering was started as the live coding sessions. See https://github.com/gerph/riscos-wimp-textrender for the details.
    * The Wimp text rendering is using a library I created a long time ago which abstracts the graphics operations into logical routines. See https://github.com/gerph/riscos-gcontext for details of the library.
* Contributed to Matthew Godbolt's TBAFS archive extraction tool, to improve some of the features. See https://github.com/mattgodbolt/tbafs.
* Released updated versions of Gareth McCaughan's `mkdrawf`/`decdrawf` tools to create and decode RISC OS Drawfiles. See https://github.com/gerph/garethmccaughan-mkdrawf for details.
    * Released under MIT license, as 32-bit safe.
    * Updated to run on 64-bit Ubuntu as well.
* Updated my `lessfilter` tooling. See https://github.com/gerph/lessfilter for details.
    * Support for listing RISC OS zip archives.
    * Improved frontmatter recognition in markdown.
    * Improved colouring in some `decaof` output.
    * Use `objdump` for 32-bit ARM ELF files.
    * Support showing Drawfiles with `decdrawf`.
        * [Screenshots](https://www.riscosopen.org/forum/forums/1/topics/20222)
        * [Video](https://asciinema.org/a/768154)
* RISC OS Build service (and shell) updated to the 0.79 version of RISC OS Pyromaniac. See https://shell.riscos.online/ and https://build.riscos.online/.
    * Added `picoc` to the tools that are present, allowing testing with simple C scripts in both 32-bit and 64-bit configurations.
* RISC OS language server for SWI calls created. See https://github.com/gerph/riscos-swi-language-server for details.
    * [Video for SublimeText: SWI help](https://share.gerph.org/s/tFGhUkZ9T5UVYsO)
    * [Video for SublimeText: Completions](https://share.gerph.org/s/jcfYJUgD02efYi5)
    * [Video for VSCode](https://share.gerph.org/s/znYmnFhyG3bZTNy)
* Port of the `wren` language to RISC OS 64. See https://github.com/gerph/riscos64-wren for details.
    * [Video](https://asciinema.org/a/762898)
* Port of `MicroQuickJS` to RISC OS 64. See https://github.com/gerph/riscos64-mquickjs for details.
    * [Video](https://asciinema.org/a/764232)
* Port of `MicroUI` to RISC OS (both architectures). See https://github.com/gerph/microui for details.
    * [Video](https://share.gerph.org/s/w8UEFpWZZcXvB4K)
* All GitHub repositories updated.
    * Automation has been added to make updating dependencies less time-consuming.
    * PR and issue templates have been added to help guide users to give better reports on issues.
* Released the UUID module in case anyone needed it. See https://github.com/gerph/riscos-uuid for details.
* Created example repository for using timed events in applications. See https://github.com/gerph/riscos-timed-events-example for details.
* Updated the build environment with the latest tooling. See https://hub.docker.com/r/gerph/riscos-build for details.
* Started sponsorship programme on GitHub. See https://github.com/sponsors/gerph for details.

The overall status of the components can be found in the [RISC OS 64 Status repository](https://github.com/gerph/riscos64-status/wiki).

You can always follow the repositories to see what’s being done. The RISC OS Status repository also has a `Discussions` section for general discussions on the progress of 64-bit work.

The sponsorship programme is something that GitHub offers to allow people to provide contributions towards the RISC OS 64 project. If you like what's being done, and want to encourage more, feel free to sponsor.

And of course, if you want to get involved in working on any of the conversion to C of components, please speak up.
